🧩 Module

Template for typical module written on Go.

💡 Idea

Define a powerful template that enables us to easily create a new Go module from scratch. Not only does it provide a starting point for our projects, but it also comes equipped with pre-configured pipelines and inventory.

🏆 Motivation

We want to start new projects quickly using best practices with a predefined structure and focusing on core ideas implementation rather than wasting time on environment configuration and copying boilerplate code.

🤼‍♂️ How to

The template saves us a lot of time and effort by streamlining the setup process, allowing us to focus on the essential aspects of our ideas.

The structure

  • .github/workflows — GitHub Actions pipelines.

Build your own module

  1. Generate a new repository from the template.
  2. Clone the repository.
  3. Update files
    • README.md,
    • docs/readme.mdx,
    • go.mod.
  4. Write your code and tests.

Contribute to the template

  1. Fork the repository.

🧩 Integration

The project uses SemVer for versioning, and it is not BC-safe through major releases. Please use go modules to manage its version.

$ go get go.octolab.org@latest
